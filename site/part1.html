Mini Knowledge Assistant

เว็บแอป "คุยกับเอกสารของคุณ" - login, อัปโหลด PDF/TXT แล้วถาม AI เกี่ยวกับเนื้อหาในเอกสาร พร้อมประวัติแชทหลายห้องแบบ ChatGPT และตัวนับ token usage แบบ real-time

Install & Setup

วิธี Docker (คำสั่งเดียว)

git clone https://github.com/Liondetnarin/trainee-knowledge-assistant.git
cd trainee-knowledge-assistant
cp .env.example .env
# แก้ .env: ตั้งค่า AI_PROVIDER, AI_API_KEY, AI_MODEL (ดู comment ใน .env.example)
docker compose up

แอปรันที่ http://localhost:3000 - ข้อมูล (SQLite + ไฟล์ที่อัปโหลด) เก็บใน Docker named volume ไม่หายเมื่อ restart · healthcheck ที่ GET /api/health

Features

Required (ครบ 5 ข้อ)

  • Login + Protected Routes (bcrypt + iron-session, กันด้วย middleware)
  • Upload PDF/TXT (validate type/size, sanitize ชื่อไฟล์, กัน path traversal)
  • Chat with AI (streaming, timeout, error handling พร้อม hint บอกวิธีแก้)
  • Chat with Uploaded File Context (chunked retrieval, ถาม-ตอบรายเอกสาร)
  • Token Usage Counter (ต่อข้อความ, ต่อบทสนทนา, รวมต่อผู้ใช้)

Bonus

  • Streaming responses (Server-Sent Events แบบ token-by-token)
  • Markdown rendering ในคำตอบ AI
  • ประวัติแชทหลายห้อง (sidebar แบบ ChatGPT - สร้าง/สลับ/ลบ)
  • Citation - คำตอบแสดงว่าใช้ chunk ไหนของเอกสารไหน
  • Rate limiting (ต่อผู้ใช้, 20 requests/นาที ที่ /api/chat)
  • Docker Compose + healthcheck
  • RAG ด้วย vector DB จริง - ไม่ทำ โดยตั้งใจ (ดูเหตุผลใน DECISIONS.md)
  • Unit tests - ยังไม่ได้ตั้ง test runner

Architecture

[Login] → iron-session cookie → middleware คุ้มครอง /chat, /upload, /api/*

[Upload PDF/TXT]
  → validate (type/size/ไม่ว่าง) → sanitize ชื่อไฟล์ → บันทึกลง disk
  → ดึงข้อความ (pdf-parse หรือ raw utf-8) → chunk (~800 ตัวอักษร, overlap 100)
  → เก็บ chunks ลง SQLite

[Chat]
  → เลือก/สร้างบทสนทนา (sidebar) → เลือกเอกสารได้ (ไม่บังคับ)
  → ดึง chunks 3 อันดับแรกที่ keyword ตรงกับคำถามมากที่สุด
  → ประกอบ system prompt → ส่งประวัติแชท + คำถาม
  → stream คำตอบกลับผ่าน SSE → บันทึกข้อความ + token usage + citation

Layering: app/api/**/route.ts (HTTP เท่านั้น) → lib/services/** (business logic) → lib/repositories/** (DB) → lib/** (utils กลาง)

Known Issues (เขียนไว้ตรง ๆ)

  • Chunking เป็น keyword-based ไม่ใช่ semantic - ดีกับคำถามข้อเท็จจริงตรง ๆ อ่อนกับคำถาม paraphrase - เหตุผลและแนวทาง upgrade อยู่ใน DECISIONS.md
  • Rate limiting เป็น in-memory instance เดียว - deploy จริงหลายเครื่องต้องใช้ Redis
  • Citation บอกระดับ chunk ไม่ใช่ประโยคเป๊ะ ๆ
  • บัญชี mock ใช้รหัสผ่านเดียวกัน - เหมาะกับ demo เท่านั้น
