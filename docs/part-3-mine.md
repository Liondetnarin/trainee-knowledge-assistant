ความคิดเห็น และแผนการของผม

ระยะเวลา 4 เดือน
5 คน
~16 สัปดาห์
ให้ตัวแปลควบคุมที่มีเหมือนกันคือ ทุกคนมี AI, Agentic IDE, AI Agent ครบทุกคนเหมือนกัน สามารถใช้ได้ไม่มี limit

สัปดาห์แรกจะประเมินงาน และวัดความรู้แบ่งจำแนกประเภท และลักษณะงานที่เหมาะสม
1. จำแนกจากความรู้ความสามารถ โดยการมอบหมายงานให้ทำ
    - มอบหมายงานเล็กๆให้ทำโดยที่เราเป็นหัวเรือใหญ่ในงานนั้น อาจเป็นงาน Vibe Coding website E-Commerce สักงาน
    - จัดตำแหน่งที่ intern ชอบ และคิดว่าตัวเองถนัด เช่น มี intern UX/UI 1คน / Full-Stack 2 คน / Front 1 / Back 1
    - แจกจ่ายงาน ระยะเวลา scope ที่ตัวเองต้องทำ แล้วลงมือทำด้วยกัน
    - สอนขั้นตอน อธิบายการทำงานร่วมกัน ถ้าใครสงสัยให้ถามในกลุ่ม หรือส่วนตัวได้เลย
    - และประเมินจากงานที่ออกมา จากความใฝ่รู้ ค้นหาข้อมูลด้วยตัวเองก่อนจนตัน และมาถามจึงสามารถสำเร็จส่วนงานของตนเองได้
    - จะได้แยก จำแนกกลุ่ม intern เพื่อเข้าใจ และสามารถมอบหมายงานต่อไปได้อย่างถูกต้อง

2. จำแนกกลุ่ม ความคิด อุปนิสัย ของแต่ละคนได้
    - เด็กฝึกงานส่วนใหญ่ที่เข้ามาสาย dev จะเก็บตัว(เป็นเหมือนกัน) และปรับตัวไม่เก่ง ต้องประเมินก่อนว่า เป็นแบบไหนเพื่อเข้าใจ และสื่อสารกันได้อย่างถูกต้อง
    - หลังจากรู้คร่าวๆจากการคุย ทำงานด้วยกัน พูดคุยเล่นต่อหลังเลิกงาน ก็จะได้พอเข้าใจนิสัยของแต่ละคนได้
    - intern ที่ถามบ่อยมากๆเพื่อให้ตัวเองเข้า หรือถามโดยที่ไม่หาข้อมูลคร่าวๆมาก่อน, intern ที่ไม่ถามเลยแต่งานไปต่อได้ หรืองานไม่เดิน
    - หลังจากเข้าใจความคิด และอุปนิสัย จะได้สามารถ เข้าหาพูดคุย และมอบหมายงานต่อไปได้อย่างถูกต้อง

3. สอนการใช้งาน AI agent ที่เราเข้าใจ หรือเทคนิคต่างๆที่เราพอจะรู้ให้ intern สามารถใช้งานได้อย่างมีประสิทธิภาพ
    - เทคนิคการใช้ IDE, AI Agent และเรียนรู้เทคนิคใหม่ๆตลอดด้วยตนเอง และจากตัว intern ด้วย
    - ให้คำแนะนำ อย่างมีเหตุมีผล และตรงกัับอุปนิสัยของคนนั้นๆ

สัปดาห์ 2-4 (เดือนแรก) Update mini project ที่ทำด้วยกันมา
1. จากการที่ได้แบ่งงาน scope งานกันแล้ว อย่างถูกต้องมากที่สุดแล้ว ก็คอยเก็บ task ของแต่ละ meeting จนจบงาน
    - บางคนบรีฟงานแล้ว ไม่ต้องบรีฟหรือถามเพิ่มก็ได้งาน task ออกมาได้อย่างดีตามงานที่ควรจะเป็น ก็จะให้ช่วยสอนเพื่อนๆคนอื่นๆ เล็กน้อยๆด้วยว่ามีเทคนิคใด มาแบ่งปัน แชร์ความรู้กันเพื่อให้สามารถทำงานได้
    - บางคนถามเพื่อความแน่ใจ จนมากเกินไป หรือไม่หาข้อมูลก่อนก็จะให้ ลองหาข้อมูลก่อน ทำแล้ว report กับตัวเองดูว่าสามารถแก้ปัญหาได้ไหม 1 2 3 4 ตามลำดับ แต่หากทำแล้วยังเจอปัญหาก็มา report และคุยกัน
    - บางคนทำงานพลาดบ่อย ไม่ถาม เขิน กลัว ไม่กล้าถาม จนงานพลาดซ้ำแล้วซ้ำอีก ก็จะคุยกันอ้อมๆก่อนกับปัญหาที่เกิด เช่น งานตรงนี้ถ้ไม่เข้าใจอะไร ลองหาข้อมูลก่อนได้เลย แต่ถ้าไม่ได้จริงๆถามพี่ได้
        - แต่ถ้ายังไม่กล้าก็จะดู และละลายพฤติกรรม เช่น พูดคุยกันตรงๆถึงผลกระทบที่เกิดขึ้น, ยอมรับความผิดพลาดของตัวเอง ทุกคนสามารถผิดพลาดได้หมดเพื่อที่จะได้รู้วิธีแก้ไข และไม่ผิดซ้ำเรื่องเดิมอีก, พูดคุยสร้างความสัมพันธ์ที่ดีที่ไม่ทำให้ intern เกร็งจนไม่กล้าคุยด้วย

2. หลังจากงาน mini project เสร็จแล้วก็จะมา meeting report ถึงส่วนต่างๆที่ทุกคนได้ทำ
    - ชมคนที่ทำดี ไม่มีทีติ หรือมีน้อย, ให้กำลังใจคนที่ทำดีรองลงมาคนอื่นๆ
    - ให้ทุกคนแสดงความคิดเห็น ข้อดีข้อเสียของตัวเองที่ได้ทำ ส่วนไหนอยากโชว์ ส่วนไหนอยากพูดถึง
    - ให้แสดงความคิดเห็นกับตัวเอง และเพื่อว่ามีส่วนไหนที่ต้องแก้ไข ปรับปรุงบ้างไหม เพื่อให้สามารถปรับปรุงได้อย่างถูกต้อง แนะนำแนวทางที่ถูกต้องให้ และเป็นกระจกของกันและกัน แต่ก็ต้องควบคุมไม่ให้มากจนทำให้บางคนให้ตรงนี้เป็นช่องที่ที่พูดไม่ดีหรือเหยียบคนอื่น เช่น สลับกันทำงานให้ส่วนที่ตนเองไม่ถนัดด้วย จะได้เป็นการพัฒนาตนเองไปในตัว และรู้ว่าตัวเองไม่ได้ไร้ที่ติจนมีอีโก้ และเปิดโอกาสให้เพื่อนที่ถนัดสอน และพูดคุยแลกเปลี่ยนความรู้กัน

สัปดาห์ที่ 5-9 (เดือนสอง)
1. วางตำแหน่งที่แต่ละคนถนัดให้เหมาะสมเพื่อจะได้ทำวงานได้อย่างมีประสิทธิภาพ
    - ทำงาน product จริงที่มีผลกระทบจริง แรงกดดันจริง โดยทำเป็น task น้อยๆ ในแต่ละส่วน เพื่อค่อยๆทำความเข้าใจ และปรับตัวได้ทีละนิด
    - แทรกด้วย project ส่วนตัวเล็กของแต่ละคน แต่ต้องไม่เพิ่มภาระหน้าที่มากเกินไปจนทำให้ไม่มีเวลาทำงานหลัก หรือรบกวนเวลา่พัก
    - meeting พูดคุยถึงปัญหา product จริง ที่เจอ แลกเปลี่ยนควาดคิดกันว่ามีปัญหาอะไรบ้าง ช่วยแก้กันอย่างไรได้บ้าง

2. ทำงาน product จริงจนสำเร็จ
    - ระหว่างงานอาจะมีปัญหา ทั้งงานช้า งานผิด ก็ต้องแก้ปัญหาด้วยวิธีที่บอกไว้ก่อนหน้า แต่ถ้าปัญหาใหม่ๆก็จะพูดคุยส่วนตัวถึงปัญหาที่เจอ และให้ทุกคนช่วยกันหาแนวทางแก้ไขไปด้วยกัน
    - ในบริบทแบบนี้ งานตั้งแต่เดือนแรกจนถึงตอนนี้จะมี กลุ่มคนสองประเภทที่จะทิ้งระยะห่างกันคือกลุ่ม หรือคนที่เก่งมากๆ จะนำโด่งไปเลย และทิ้งกลุ่มหรือคมที่ช้ากว่ารั้งท้าย ต้องพูดคุยเป็นการส่วนตัวหรือประชุมรวม เพราะส่วนมากจะรู้ตัวเองอยู่แล้วว่าเป็นกลุ่มไหน และจะรู้สึก กดดัน หรือยอมแพ้ หรืออิจฉา มีอีโก้ หรืออื่นๆ
        - ต้องทำให้รู้ว่าเราทีมเดียวกัน เราพวกเดียวกันทั้งหมด ถึงในความเป็นจริงคนที่ดีที่สุดจะสามารถนำตัวเองไปถึงฝั่งได้เร็วโดยที่ไม่ต้องสนใจคนข้างหลังก็ได้ ซึ่งเป็นเรื่องดีส่วนหนึ่งเพราะตนเองสามารถพึ่งพาตัวเองได้อย่างเต็มร้อยและบรรลุเป้าหมายได้อย่างดี แต่ก็ต้องคอยย้ำทุกครั้งว่าการที่เราผ่านจุดนั้นมาแล้ว เข้าใจแล้ว ถ้าเราแบ่งปันความรู้ความเข้าใจสอนคนอื่นได้ คนที่เราสอนก็จะสามารถทำแบบเดียวกันกับเราได้ และสอนคนอื่นๆต่อไป
        - คนที่ช้ากว่า เราก็ต้องกระตุ้นด้วยหลายๆวิธี กับคนหลายๆประเภท เช่นชอบความท้าทาย กดดันเพราะถีบตัวเองได้ดี หรือคนที่ต้องมีแรงผลักดันจากคนอื่น ไม่อยากกดดัน กระตุ้นให้เหมาะกับอุปนิสัยอย่างถูกต้อง และคอยย้ำเสมอว่า ที่เราช้าไม่ใช้เพราะเราไม่เก่ง ไม่ใช่เพราะเราไม่มีความสามารถ หรือไม่ใช่เพราะเราไม่พยายาม แต่เพราแนวทางที่เราเลือกอาจจะไม่ถูก ให้รับฟังความรู้จากคนอื่นๆ หรือมั่นหาความรู้ตลอดเวลาเพื่อพัฒนามากขึ้น ยอมรับความผิดพลาดของตัวเอง ถอยออกมาหนึ่งก้าวเพื่อมองภาพรวมได้ชัดเจนขึ้นจะได้ัก้าวเดินทางใหม่ได้อย่างถูกต้องและเหมาะสมที่สุด

3. หลังจากงานเสร็จ และ project ส่วนตัวเสร็จแล้วก็จะมา meeting report ถึงส่วนต่างๆที่ทุกคนได้ทำ
    - ชมคนที่ทำดี ไม่มีทีติ หรือมีน้อย, ให้กำลังใจคนที่ทำดีรองลงมาคนอื่นๆ
    - ให้ทุกคนแสดงความคิดเห็น ข้อดีข้อเสียของตัวเองที่ได้ทำ ส่วนไหนอยากโชว์ ส่วนไหนอยากพูดถึง
    - ให้แสดงความคิดเห็นกับตัวเอง และเพื่อว่ามีส่วนไหนที่ต้องแก้ไข ปรับปรุงบ้างไหม เพื่อให้สามารถปรับปรุงได้อย่างถูกต้อง

สัปดาห์ที่ 10-16 (เดือนสามถึงเดือนสุดท้าย)
1. หลังจากที่ทุกคนได้ทำหน้าที่ชองตัวเองได้อย่างถูกต้องและเหมาะสมแล้ว ก็คอยเก็บ task, report อย่างสม่ำเสมอ
    - task งานที่ได้รับมอบหมาย 
    - report งานที่ทำ ปัญหา แนวทาง อื่นๆ
    - คอยดูข้อผิดพลาดที่อาจเกิดขึ้น และรีบเตือนก่อนรุนแรง
2. report งานภาพรวมตั้งแต่วันแรกว่าตัวเองได้พัฒนาอะไรบ้างจากเดิม แล้วมีข้อติอะไรตรงไหนที่ต้องปรับปรุง
