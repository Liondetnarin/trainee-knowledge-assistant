# Part 2 — Video Notes (Bullet เท่านั้น — อย่าอ่านทั้งหมด)

> Scenario: ลูกค้า ABC โกร wrath — ระบบใช้ไม่ได้ตอน demo  
> ความจริง: credential ผิด + พี่ reset password แต่ลืมแจ้ง

## โครง 3 นาที

### 1. รับสาย
- [ ] หายใจ / ฟังให้จบ ไม่ interrupt
- [ ] "เข้าใจครับ/ค่ะ ว่าตอนนี้เครียดมาก โดยเฉพาะตอน demo"

### 2. Empathy + ซื้อเวลา
- [ ] ขอโทษในนามบริษัท (ไม่โยนความผิดทันที)
- [ ] "ขอเช็คให้ละเอียด 10–15 นาที แล้วโทรกลับได้ไหมครับ"

### 3. แก้ปัญหา
- [ ] เช็คระบบ → พบว่า technical OK
- [ ] คุยพี่ในทีม → รู้ว่า reset password เมื่อวาน
- [ ] โทรกลับ → แนะ reset/รหัสใหม่ step-by-step

### 4. Follow-up
- [ ] email สรุปเหตุการณ์
- [ ] prevention: แจ้งลูกค้าทุกครั้งที่ reset + checklist
- [ ] คุยพี่แบบ private หลังจบ (ไม่ด่าหน้าลูกค้า)

## Bonus ที่จะพยายามใส่
- [ ] Root cause: process ไม่มี confirmation หลัง reset
- [ ] Goodwill: extended support / training session สั้น ๆ
