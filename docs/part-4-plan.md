## Outline คำถามที่ 1: 3 วันแรก (10 คะแนน, เป้า ~250–400 คำ)

คำถามช่วยคิด (ตอบด้วยมุมมองตัวเอง ไม่ใช่ท่องมา):

- **Priority**: อะไรทำก่อน? แนวคิดทั่วไปที่ปลอดภัย = read-only ก่อน (log, monitoring, backup/snapshot) → เข้าใจ architecture → ค่อยแตะของจริง ทำไมถึงเรียงแบบนี้ในมุมคุณ?
- **Day 1 / Day 2 / Day 3**: แต่ละวันโฟกัสอะไร? (เช่น Day 1 = อ่าน/สัมภาษณ์คนที่ยังพอรู้เรื่อง + snapshot ปัจจุบันไว้ก่อนแตะอะไร, Day 2 = ขุด root cause ของความช้า, Day 3 = สรุป findings + risk list)
- **ใช้ AI ยังไง**: ต้องมี prompt ตัวอย่าง 1–2 อันที่ realistic เช่น "ช่วยอธิบาย pattern ของ PHP framework เก่าตัวนี้จาก code นี้" หรือ "ช่วย list สิ่งที่ต้องเช็คก่อนแตะ production server ที่ไม่มีเอกสาร" — เขียนเองว่าจะถามอะไรจริง ๆ
- **ถ้าไม่รู้ จะถามใคร**: senior dev ในทีม? ลูกค้า (business context)? sysadmin (infra access)?
- **เรื่องที่กังวล**: กลัวรัน command แล้วพัง prod, กลัวไม่มี rollback plan
- **จบ 3 วัน deliver อะไร**: discovery doc สั้น ๆ + risk list + priority ของสิ่งที่ต้องทำต่อ (ไม่ใช่ fix เสร็จ)

## Outline คำถามที่ 2: ไฟล์ PHP obfuscate ใน images/ (10 คะแนน, เป้า ~150–300 คำ)

คำถามช่วยคิด:

- **คิดว่าคืออะไร (ยังไม่ฟันธง)**: อาจเป็น webshell/malware หลงเหลือจาก incident เก่า หรืออาจเป็นโค้ด legit ที่ obfuscate ไว้ด้วยเหตุผลอื่น (license, ป้องกัน copy) — สำคัญคือ "ไม่ฟันธง"
- **คำถามที่อยากถามก่อนตัดสินใจ**: ใครสร้างไฟล์นี้/เมื่อไหร่? มี access log ไหมว่าถูกเรียกใช้บ้างหรือเปล่า? เคยมี incident ด้าน security มาก่อนไหม?
- **จะทำยังไง — ทำไม**: แนวคิดที่ปลอดภัย = **อย่าลบทันที** (เสียหลักฐานถ้าเป็น webshell) → quarantine/snapshot ไฟล์ไว้ก่อน → scan ด้วย security tool → ห้ามรันไฟล์บน prod เด็ดขาด
- **อะไรต้องเช็คเพิ่ม**: server log, ไฟล์อื่นที่ modify time ใกล้เคียงกัน, network traffic ผิดปกติ
- **ใครควรรู้เรื่องนี้**: พี่ในทีม + อาจต้องแจ้งลูกค้า/security team ถ้ายืนยันว่าเป็น malware จริง

## Red Flags ที่ต้องเลี่ยง (จาก assignment-spec + rubric)
- มั่นใจว่า "แก้จบได้ใน 3 วัน" — ไม่ realistic กับ legacy 10 ปี
- "ลบไฟล์แปลกทิ้งเลย" โดยไม่มีเหตุผลประกอบ
- ไม่มี priority — ทำทุกอย่างพร้อมกัน
- ทั้งคำตอบไม่มีคำถามกลับเลย
- เนื้อหาเป็น AI 100% — ขาดมุมมองส่วนตัว/adjustment
