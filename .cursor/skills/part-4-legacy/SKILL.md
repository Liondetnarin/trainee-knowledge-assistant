# Part 4 — Legacy System Thinking

## ข้อกำหนด
- ความยาว: **400–800 คำ** (ลึกดีกว่ายาวผิวเผิน)
- ไฟล์: `docs/part-4-legacy.md`
- โทน: **คุยกับพี่ในทีม** — ไม่เป็นทางการเกิน

## Context สรุป
- เว็บลูกค้า 10 ปี, ไม่มี docs/test, dev เก่าลาหมด
- 3 Windows Server, PHP เก่า, deploy copy ไฟล์
- ปัญหา: ช้า, ไฟล์แปลกใน server, อยาก deploy feature ใหม่
- เวลา: 2 สัปดาห์ discovery — **ไม่ต้องแก้จบ**

## คำถามที่ 1: 3 วันแรก (10 คะแนน)
ต้องมี:
- [ ] ลำดับ priority (อะไรก่อน-หลัง + เพราะ)
- [ ] ใช้ AI ยังไง + prompt ตัวอย่าง 1–2 อัน
- [ ] ถ้าไม่รู้ ถามใคร/ถามอะไร
- [ ] เรื่องที่กังวล (เช่น กลัวพัง prod)
- [ ] จบ 3 วัน deliver อะไรให้พี่

## คำถามที่ 2: ไฟล์ .php obfuscate ใน images/ (10 คะแนน)
ต้องมี:
- [ ] คิดว่าคืออะไร (ยังไม่ฟันธง)
- [ ] คำถามก่อนตัดสินใจ
- [ ] ลบ / เก็บ / ทำอย่างอื่น — ทำไม
- [ ] อะไรต้องเช็คเพิ่ม
- [ ] ใครควรรู้

## Red Flags (หลีกเลี่ยง)
- "แก้จบ 3 วัน" — ไม่ realistic
- "ลบทิ้งเลย" โดยไม่มีเหตุผล
- ไม่มี priority / ไม่มีคำถามกลับ
- AI 100% ไม่มีมุมมองตัวเอง
